About the Data

We initially decided to use the kaggle Covid-19 tweeet dataset. However, during our initial exploratory data analysis, we discovered that many of the tweet texts of the dataset were truncated. As we wanted to extract sentiment scores and topics from the unstructured fulltext content of the tweets, we needed access to the tweet texts to their entirety. We decided to retain the original columns of the Kaggle dataset and acquire the tweet text ourselves using the Twitter API. For the final dataset, we used the twitter API to collect a customized set of tweets that ranges from Jan. 27th to Sept. 30th, which represents a wider time range. The total number of tweets we collected is 91448, and we kept all the original columns of the Kaggle dataset.

Feature Engineering

For additional information, we extracted the sentiment scores of each tweet using three different packages: NLTK, Transformers, and TextBlob. We finalized on the Transformer sentiment scores for all our visualizations. We extracted significant phrases or topics from each unstructured Tweet text. This topic extraction process provides us with methods to organize, understand and summarize large collections of highly unstructured textual information stored in the covid-19 twitter text. The topics enable us to discover latent topical patterns that are present across the dataset. We have used a Python library called RAKE, which stands for Rapid Automatic Keyword Extraction. The algorithm behind the RAKE implementation of topic modeling is based on the “Text Mining Applications and Theory” book by Michael W. Berry. Furthermore, with our sentiment scores and sentiment labels, we are able to map the extracted set of topics to sentiment score range clusters. We define the sentiment score range clusters as a spectrum of sentiments, from ‘Negative Sentiment 90-100’ to ‘Positive Sentiment 90-100’. All other cluster ranges are mapped in between the aforementioned lower bound and upper bound clusters.
